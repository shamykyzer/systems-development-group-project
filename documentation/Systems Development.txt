Project Aims The aim of this project is to design and implement a web-based graphical
application that integrates a machine learning-driven sales forecasting system
using Meta’s Prophet for five cafés. The system will generate short-term
demand predictions for food and beverage items to optimise purchasing
decisions, with the objective of reducing food waste and minimising financial
loss.

Project Objectives
➔

Develop and release a web-based sales application using a lightweight framework (React.js) with core features including a
dashboard, sales history, and parameter tweaking. Target completion: April 2026.

➔

Implement and train a sales forecasting AI model achieving ≥85% accuracy on historical sales data, with inference executed
within reasonable response times. Deliver a working prototype by April 2026.

➔

Design and deploy an accessible dashboard displaying historical sales data, compliant with WCAG 2.1 AA accessibility
standards. Completion target: April 2026.

➔

Conduct structured application testing, including invalid input handling and stress testing, ensuring ≥95% of test cases pass
with no critical defects. Complete testing by April 2026.

➔

Maintain code readability and version control standards, requiring descriptive comments on all functions and complex logic.
All pull requests must be reviewed and approved by at least two team members before merging.

Technology Stack

-

Frontend: React.js with Tailwind CSS for a responsive, lightweight user
interface.
Backend: Python with SQLite for handling application logic and data storage.
Deployment: Docker used to containerised components for consistent
execution.
Version Control: GitHub used to manage collaboration and prevent conflicts.

This stack was selected to enable rapid development, modular design, and
reproducible deployment.

AI Model Implementation
-

Implement Meta’s Prophet to develop a lightweight and efficient time-series
forecasting model for sales prediction.
Utilise Pandas and NumPy for data preprocessing, transformation, and forecast
visualisation.
Enable structured access to project .csv datasets, allowing the model to ingest,
train, and generate accurate predictions.
Refine and standardise forecast visualisations to ensure consistent styling and
colour schemes across the application.

Dashboard Design
-

Design an accessible, easy-to-read dashboard for displaying historical sales
data.
Standardise colour schemes, fonts, and text sizes for visual consistency and
ethical accessibility.
Ensure use of open-source fonts and compliant styling to address copyright
concerns.
Provide quick access to historical sales, e.g., a dedicated folder of
charts/images.
Implement concise, consistent charts and filtering options to allow users to
focus on relevant data.

Testing & Quality Assurance
-

Implement GitHub Actions to prevent broken code from being merged into the
main branch.
Develop a structured testing plan to maintain a consistent, bug-free
application.
Maintain a testing log to track test execution and reproduce bugs effectively.
Generate varied datasets to verify proper error handling and full functionality.
Conduct UI testing across different screen sizes to prevent layout or boundary
issues.

Code Readability & Version Control

-

-

Maintain regular GitHub commits with a review rule requiring approval from at
least two team members before merging.
Write clear, descriptive comments and follow consistent spacing and
indentation, especially in HTML and frontend code, to improve readability and
ease debugging.
Schedule regular code audits and team check-ins to ensure everyone
understands the codebase and changes are properly tracked.
Enforce coding standards and documentation practices to keep the project
maintainable as it grows.

Requirements - Functional
-

Development of a basic frontend that provides a simple yet elegant GUI
allowing the end user to interact with and understand the information.
Use of an algorithm that can predict the sales of croissants, Cappuccinos, and
Americanos over the upcoming weeks.
Use of graphs to display current and predicted sales.
Allow the user to change training periods and explore predictions with
different lengths of training data.
Basic login system for data security to prevent others from accessing the
sensitive data.

Requirements - Non-Functional
-

-

Easy to use and access application, with well organised graphics and the
ability to function on multiple screen sizes, and a consistent theme throughout.
After use, the algorithm should provide well designed and laid out graphs that
are easy to understand and interpret, and should display correctly on multiple
screen sizes.
Predicted sales should be output in a well laid out, easy to read manner that
makes it easier to understand for the company using the algorithm.
App can also have extra security features to help protect against data leaks
and protect the sales information of the company.
Algorithm should work in a reasonable time scale, so that it can be used easily
to bring up the sales predictions that are needed.
